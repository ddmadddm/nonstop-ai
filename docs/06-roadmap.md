# 06. 개발 로드맵

단계별로 "동작하는 작은 시스템 → 확장" 순서. 각 단계 끝에 실제로 쓸 수 있는 상태를 목표로 한다.

## 0단계 — 환경 셋업
- [ ] Next.js(App Router, TypeScript) 프로젝트 생성 (`C:\Projects\NONSTOP-System`)
- [ ] Supabase 프로젝트 생성, `.env` 연결
- [ ] `pgvector` / `pg_trgm` 확장 + `0001_init.sql` 마이그레이션 적용
- [ ] Supabase Auth 로그인 + `(dashboard)` 보호 라우트
- [ ] 기본 레이아웃·네비게이션

## 1단계 — 상담 저장 + 조회 (수집 코어) ★MVP 핵심
- [ ] `POST /api/kakao/webhook` 수신·저장(멱등) — *카카오 스펙 확정 전에는 모의 페이로드로 개발*
- [ ] customers/conversations/messages 저장 로직
- [ ] 상담 목록 `/conversations` (필터·정렬)
- [ ] 상담 상세 `/conversations/[id]` 메시지 스레드
- [ ] `POST /api/kakao/send` 상담원 발송 + 메시지 기록
> 이 단계만으로 "카카오 상담을 저장하고 콘솔에서 응대" 가능.

## 2단계 — 상담유형 자동분류 + FAQ
- [ ] 상담유형 관리 `/categories` (대/소분류)
- [ ] FAQ 관리 `/faqs` (CRUD + 임베딩 생성)
- [ ] 수신 메시지 자동분류(`/api/ai/classify`) → 대화 유형 갱신
- [ ] 수동 재분류 UI

## 3단계 — AI 답변 생성 (RAG) ★프로젝트 목표
- [ ] 임베딩 파이프라인(메시지/FAQ)
- [ ] `match_faqs` RPC + 답변 초안 `/api/ai/draft`
- [ ] 상담 상세에 AI 초안 패널(채택/수정/폐기)
- [ ] `ai_responses` 기록 + 참고 FAQ 노출

## 4단계 — 상담이력 검색 + 거래처
- [ ] 거래처 관리 `/clients`, 고객-거래처 매칭
- [ ] 거래처 상세 상담이력 `/clients/[id]`
- [ ] 검색 `/search` (키워드 → 하이브리드)

## 5단계 — 운영 고도화
- [ ] 대시보드 통계(유형별·기간별·처리율)
- [ ] 비동기 큐(pgmq/QStash)로 후속 처리 분리
- [ ] RLS 정교화, 감사 로그
- [ ] AI 채택률·수정 패턴 분석 → FAQ 보강 루프

## 이후 — 도메인 확장
같은 코드베이스/DB 위에 모듈 추가:
- 접수센터 · 배차센터 · 기사센터 · 정산센터
- 상담에서 발생한 건을 접수→배차로 넘기는 연계 설계(공통 `clients`/`customers` 재사용).

2. **Supabase 프로젝트 / OpenAI API 키** — 0단계 시작 전 필요.
