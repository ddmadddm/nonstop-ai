# 07. 상담유형 체계 (퀵/화물 배차 중개)

AI 자동분류·FAQ·통계의 기준이 되는 상담유형 트리. **대분류 → 소분류** 2단계.
업무 구조는 [08 도메인 모델](08-domain-model.md) 참고. 실제 채널톡 데이터 기반.

## 분류 설계 원칙
1. **상호 배타적**으로 — 한 메시지가 두 유형에 걸치지 않게.
2. **소분류 5개 내외**.
3. `description`은 AI 분류 프롬프트에 그대로 사용 → 판단 근거가 드러나게.
4. 한 대화(상담) 안에 접수→배차→운임이 순차로 오므로, **메시지 단위 분류**도 고려(대화는 마지막/주요 유형으로 표기).

## 트리

### 1. 접수/오더 — 거래처가 운송을 요청
| 소분류 | 설명(분류 기준) |
|--------|------------------|
| 신규 오더요청 | "퀵요청드립니다" 등 새 운송 요청 |
| 경유/다중배송 | 경유지 여러 곳, 다중 하차 |
| 시간지정/즉시 | "즉시입차", "11시" 등 시간 조건 |
| 접수 확인문의 | 가능 여부·경유 순서·차량 가능 등 접수 관련 질문 |

### 2. 배차 — 기사 매칭/진행
| 소분류 | 설명 |
|--------|------|
| 배차 진행 | 기사 배정 진행 중 안내 |
| 배차완료 안내 | 배차정보(기사·차량·운임) 전달 |
| 재배차 | 기사 이탈·취소로 다시 배차 ("기사님 빠지셔서 재배차 중") |
| 차량 매칭문의 | 다마스/오토/라보/1톤 등 차종 가능 여부 |

### 3. 운임/요금
| 소분류 | 설명 |
|--------|------|
| 운임 문의/견적 | 예상 운임 문의 |
| 운임 안내 | "운임 40,000원(공급가)" 등 확정 안내 |
| 추가요금 | 경유·대기·할증 등 추가 비용 |

### 4. 운행/현황
| 소분류 | 설명 |
|--------|------|
| 기사 위치/현황 | 지도 공유, 기사 현재 위치 ("[기사▶출] N km") |
| 상차/픽업 확인 | 출발지 도착·상차 여부 |
| 도착예정/지연 | 도착 시간·지연 안내 |
| 운행완료 | 하차·완료 확인 |

### 5. 변경/취소
| 소분류 | 설명 |
|--------|------|
| 오더 변경 | 주소·시간·물품·차량 변경 |
| 오더 취소 | 접수 건 취소 |

### 6. 기사 관련
| 소분류 | 설명 |
|--------|------|
| 기사 연락처 | 담당 기사 연락처/연결 |
| 기사 클레임 | 기사 응대·지각·태도 불만 |

### 7. 정산/세금계산서
| 소분류 | 설명 |
|--------|------|
| 정산 내역 | 월 정산·운임 내역 확인 |
| 세금계산서 | 발행·재발행·수정 |
| 미수/미납 | 미수금·미납 |

### 8. 거래처/계약
| 소분류 | 설명 |
|--------|------|
| 신규 거래 문의 | 거래처 등록·이용 시작 |
| 단가/계약 | 단가표·운임 계약 조건 |

### 9. 클레임/사고
| 소분류 | 설명 |
|--------|------|
| 지연 클레임 | 늦은 도착으로 인한 불만/보상 |
| 파손/분실 | 물품 파손·분실 |
| 오배송 | 잘못된 하차·오배송 |

### 10. 기타/일반
| 소분류 | 설명 |
|--------|------|
| 이용 방법 | 요청 방법·양식 안내 |
| 영업/운영 | 영업시간·운영 정책 |
| 기타 | 위 어디에도 안 맞는 경우 |

## DB 반영
- [data/categories.seed.csv](../data/categories.seed.csv)로 정리 → `categories` 임포트.
- AI 분류는 소분류 단위, 통계는 대분류로 집계.

## 확정 시 체크
- [ ] 실제 업무 유형이 맞는지(차종, 운임 표현, 경유 처리 등)
- [ ] 빠진 유형(예: 왕복·합짐·반품 등 특수 운송이 있으면 추가)
- [ ] 각 `description`이 분류 기준으로 충분히 명확한지
